# 多进程、多线程

## 进程和线程区别

```text
进程:
  资源分配的基本单位。
  进程之间地址空间相互独立。
  稳定性好，但切换和通信成本较高。

线程:
  CPU 调度的基本单位。
  同一进程内线程共享地址空间、文件描述符等资源。
  通信方便，但要处理锁、竞态、死锁等问题。
```

## fork 注意点

`fork()` 后父子进程都会从 fork 返回处继续执行。父进程拿到子进程 pid，子进程拿到 0。

## pthread 注意点

多线程访问共享变量时要加锁，否则会发生数据竞争。线程退出前要 `pthread_join`，避免资源泄漏。

## 面试口述

进程隔离性更好，线程共享资源更轻量。多线程主要难点是同步、互斥、死锁和生命周期管理。
