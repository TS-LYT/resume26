# C/C++ 宏定义、位操作、位域

## 宏定义常见用途

```text
1. 常量定义
2. 条件编译
3. 日志宏
4. 位操作
5. 简单函数式宏
```

## 宏和函数的区别

宏是预处理阶段文本替换，没有类型检查；函数有类型检查，也更安全。复杂逻辑优先用函数或 `static inline`。

## 位域 bit-field

位域可以让结构体成员按 bit 存储，常用于协议头、寄存器映射等场景。

注意：

```text
1. 位域布局和大小端、编译器实现有关。
2. 跨平台协议解析不建议直接依赖 C 位域内存布局。
3. 操作硬件寄存器时要结合芯片手册和编译器行为。
```

## 面试口述

宏没有类型检查，容易因为副作用出错，例如 `SQUARE(i++)`。位操作常用于状态标志，位域可以节省空间，但跨平台要谨慎。
