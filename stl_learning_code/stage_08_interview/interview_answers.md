# STL 面试题口述答案

## 1. vector 底层是什么？

vector 底层是动态数组，内存连续，支持 O(1) 下标随机访问。尾部插入通常很快，但容量不够时会扩容。

## 2. vector 和普通数组区别？

普通数组大小固定，接口少；vector 大小可以动态增长，提供 `push_back`、`size`、`at`、`clear` 等接口，使用更方便。

## 3. vector 和 array 区别？

array 是固定大小数组，大小编译期确定，不能扩容；vector 是动态数组，可以运行时增长，但可能涉及动态内存分配。

## 4. vector 和 list 区别？

vector 内存连续，随机访问快，尾部插入常用；list 是双向链表，内存不连续，不支持下标访问，但在已找到位置时插入删除快。

## 5. list 底层是什么？

list 底层是双向链表，每个节点除了数据，还有前后指针。它不适合频繁随机访问。

## 6. queue 和 stack 区别？

queue 是先进先出，适合任务队列、消息队列、网络包队列；stack 是后进先出，适合括号匹配、表达式解析、撤销操作。

## 7. deque 和 vector 区别？

vector 通常是一整块连续内存，适合尾部插入和随机访问；deque 是双端队列，通常是分段连续，适合头尾都频繁插入删除。

## 8. reserve 和 resize 区别？

`reserve` 只改变容量 capacity，不改变元素个数 size；`resize` 会改变 size，扩大时会创建新元素，缩小时会删除尾部元素。

## 9. size 和 capacity 区别？

size 是当前实际元素个数；capacity 是当前已分配空间最多能放多少元素。capacity 通常大于或等于 size。

## 10. vector 扩容会发生什么？

容量不够时，vector 会申请更大的连续内存，把旧元素拷贝或移动过去，再释放旧内存。扩容后原来的指针、引用、迭代器可能失效。

## 11. vector 迭代器什么时候失效？

触发扩容时，所有迭代器、指针、引用都可能失效。插入或删除元素时，插入点或删除点之后的迭代器也可能失效。

## 12. 嵌入式中使用 STL 要注意什么？

要注意动态内存分配、实时性、异常、内存碎片和最坏耗时。数据量固定时优先考虑 `array`；必须用 `vector` 时可以提前 `reserve`，避免关键路径上频繁扩容。
